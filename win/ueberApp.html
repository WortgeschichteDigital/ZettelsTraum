Version 0.4.0-next10. April 2019

Verbesserungen

  • Dropdown-Menü: Immer dann, wenn der Inhalt eines Dropdown-Menüs unterhalb des Textfeldes nicht komplett angezeigt werden kann, wird das Menü automatisch über dem Textfeld geöffnet.

Version 0.3.010. April 2019

Neue Funktionen

  • Belegzettel: Mit dem Tastaturkürzel Strg + U kann jetzt zwischen Formular- und Leseansicht gewechselt werden.
  • Belegzettel: In der Leseansicht gibt es nun eine Funktion zum Aus- bzw. Einblenden der Silbentrennung und Seitenumbrüche. Diese Einstellung wird gespeichert und beim Neustart des Programms wiederhergestellt. Sie ist auch über das Tastaturkürzel Strg + T zu erreichen.

Verbesserungen

  • Das Tastaturkürzel Strg + S und die Funktion Kartei > Speichern rufen den Speicherbefehl nun auch für das Notizen-Fenster und den Belegzettel auf. Verläuft das Speichern der Notizen bzw. des Belegs fehlerfrei, wird direkt im Anschluss die Kartei gespeichert.
  • Meldungen, dass keine Änderungen vorgenommen wurden und ein Speichern unnötig sei, wurden entfernt.
  • Belegliste: Die Maximalbreite der Absätze wurde leicht erhöht.
  • DTA-Import: Beim Import mehrerer Seiten wird der Seiten- oder Spaltenumbruch im Texte nun markiert. Angegeben wird immer die Nummer der folgenden Seite/Spalte, und zwar nach diesem Schema: "[:SEITE:]". Diese Markierung der Seitenangabe wird mit den Schaltern, die bisher nur zum Ein- bzw. Ausblenden der Silbentrennung dienten, ein- bzw. ausgeblendet.
  • Belegzettel: Die Maximalbreite des Belegzettels wurde stark erhöht.
  • Dropdown-Menü: Einträge, die mehr als 65 Zeichen lang sind, haben jetzt immer einen Tooltip.
  • Dropdown-Menü: In mehrzeiligen Feldern werden auch dann sinnvolle Vorschläge geliefert, wenn bereits ein Wert im Textfeld steht.
  • Dropdown-Menü: Erscheint das Dropdown-Menü während der Texteingabe, werden bereits im Feld vorhandene Werte nicht mehr zum Hinzufügen angeboten.
  • Dropdown-Menü: Wird infolge einer Texteingabe ein Wert aus einer gefilterten Vorschlagsliste ausgewählt, wird automatisch die Zeile überschrieben, in der begonnen wurde zu tippen.
  • Dropdown-Menü: Versehntliches Einfügen von Leerzeilen zwischen den Werten wird in mehrzeiligen Feldern von nun an vermieden.

Behobene Fehler

  • DTA-Import: Der Import der letzten Seite eines Werks war mitunter nur unvollständig.
  • Belegzettel: Wenn die Felder mit den Feldüberschriften sehr schmal wurden, konnten sich Text und Kopier-Icon unter Umständen überlappen.
  • Notizen-Fenster: Unter Umständen wurde man zweimal gefragt, ob die Notizen gelöscht werden sollen.
  • Dropdown-Menü: Unter Windows konnten Menü-Einträge nicht mit der Maus ausgewählt werden.
  • Dropdown-Menü: Die Dialoge waren dysfunktional, falls in einem der Datenfelder des Belegs bereits Text stand.
  • Dropdown-Menü: Wurden im Belegzettel die Felder Bedeutung oder Textsorte über das Dropdown-Menü gefüllt, wurden die Werte beim Speichern nicht übernommen.
Version 0.2.09. April 2019

Verbesserungen

  • Suchfunktion: Das Finden von Wörtern mit Sonderzeichen (ẽ und ñ) wurde verbessert.
  • Belegtext: Beim Kopieren wird die Quellenangabe automatisch mitkopiert. Dies betrifft alle Kopierfunktionen: das Kopieren via Icon und das Kopieren einer Textauswahl via Kontextmenü; beides sowohl in der Belegliste als auch im Belegzettel.
  • Belegzettel: Die Texttools erkennen illegales Nesting jetzt in mehr Fällen als zuvor.

Behobene Fehler

  • Belegzettel: Das Einfügen von Text im Belegfeld durch Klick auf das Einfüge-Icon funktionierte nicht.
  • Belegzettel: Beim Einfügen von Text im Belegfeld durch Klick auf das Einfüge-Icon wurden einfache Zeilenumbrüche im HTML-Quelltext ignoriert.
  • Belegzettel: Ein Hinweis auf illegales Nesting erschien nicht in jedem Fall.
