Changelog

Version 0.4.0-beta10. April 2019

Neue Funktionen

  • Es gibt nun ein separates Fenster, in dem die bisher gesammelten Bedeutungen zum jeweiligen Wort in einer Baumstruktur angezeigt werden. Geöffnet werden kann das Fenster über das Menü (Kartei > Bedeutungen), das Tastaturkürzel Strg + ⇧ + B oder das Icon in der Quick-Access-Bar.

    Die Zählung der Bedeutungsebenen beginnt vorerst mit römischen Ziffern (I., II. usw.). Dies wird sich anpassen lassen, sobald die zentrale Bedeutungsfunktion programmiert ist.

    Durch Klick auf eine der Bedeutungen kann diese in eine gerade geöffnete Karteikarte im Hauptfenster eingetragen werden. Steht dort im Textfeld Bedeutung schon ein Wert wird dieser keinesfalls automatisch, sondern allenfalls auf Nachfrage überschrieben.

    Der Bedeutungsbaum kann über das Druck-Icon in der Überschrift ausgedruckt werden. Dabei wird die Druckfunktion des Hauptfensters verwendet (s. u.).

  • Druckfunktion: Karteikarten können nun ausgedruckt werden. Die Funktion steht sowohl für einzelne Karteikarten als auch für die komplette Belegliste zur Verfügung. Beim Druck einer Belegliste werden jene Belege gedruckt, die aktuell in der Liste erscheinen.

    Aufgerufen werden kann die Druckfunktion über das Druck-Icon im Kopf der Belegliste bzw. Karteikarte. Alternativ lässt sie sich auch über das Tastaturkürzel Strg + P starten (wenn Belegliste oder Karteikarte nicht durch ein anderes Fenster verdeckt werden).

    Wird die Druckfunktion aufgerufen, erscheint zunächst ein Vorschaufenster. In diesem kann der Druck gestartet oder der komplette Inhalt des Vorschaufensters in die Zwischenablage kopiert werden.

Verbesserungen

  • Die Breite der Filterleiste passt sich dynamisch an die Breite des Programmfensters an. Sie nimmt nun 40 % des verfügbaren Platzes ein, jedoch mindestens 300 Pixel und höchsten 600 Pixel, was dem Doppelten der bisherigen fixen Breite entspricht.
  • Anstelle von DejaVu wird als Schriftart fortan Noto benutzt. (Einige im DTA häufig verwendete Glyphen sind in DejaVu nicht vorhanden.)
  • Karteikarte: Die Textfelder Bedeutung und Textsorte können auf Wunsch ohne Nachfrage aus einem Dropdown-Menü heraus ergänzt werden. Eine Option dafür findet sich in Programm > Einstellungen > Karteikarte. Die Option lässt sich auch über das Meldungsfenster aktivieren, das erscheint, wenn ein Textfeld bereits gefüllt ist.
  • Dropdown-Menü: Immer dann, wenn der Inhalt eines Dropdown-Menüs unterhalb des Textfeldes nicht komplett angezeigt werden kann, wird das Menü automatisch über dem Textfeld geöffnet.
  • Anhänge und Redaktion: Nach dem Verschieben eines Eintrags wird das Pfeil-Icon des betreffenden Eintrags fokussiert. So kann ein Eintrag mithilfe von ↵ schneller und leichter über mehrere Positionen nach oben verschoben werden.
  • Kartei > Zuletzt verwendet: Bisher gab es nur für die ersten sechs Einträge Tastaturkürzel. Ab sofort gibt es Kürzel bis Eintrag Nummer neun.

Behobene Fehler

  • Viele Funktionen des Hauptfensters waren auch aus den Nebenfenstern (Handbuch, Dokumentation usw.) heraus aufzurufen, wenn das zugehörige Tastaturkürzel gedrückt wurde. Dies könnte unerwünschte Effekte haben und wird fortan unterbunden.
  • Filterleiste: Die Darstellung kompliziert verschachtelter Bedeutungsbäume war mitunter grob falsch.
  • Karteikarte: In der Leseansicht wurde die Markierung mit Sternen fehlerhaft dargestellt.
  • Dropdown-Menü: Wurde mehrmals hintereinander auf denselben Schalter gedrückt, um ein Dropdown-Menü zu öffnen, funktionierte das Öffnen des Dropdowns via Tastaturkürzel mitunter nicht mehr.
  • Dropdown-Menü: In mehrzeiligen Feldern war der Pfeil zum Aufklappen des Menüs nicht mehr richtig zu sehen, wenn das Feld so groß geworden war, dass es eine Scrollleiste bekam.
  • Das Formular zur Eingabe neuer Belege bzw. dessen Leseansicht wird nun durchgehend als Karteikarte und nicht mehr als Belegzettel bezeichnet.
Version 0.3.010. April 2019

Neue Funktionen

  • Karteikarte: Mit dem Tastaturkürzel Strg + U kann jetzt zwischen Formular- und Leseansicht gewechselt werden.
  • Karteikarte: In der Leseansicht gibt es nun eine Funktion zum Aus- bzw. Einblenden der Silbentrennung und Seitenumbrüche. Diese Einstellung wird gespeichert und beim Neustart des Programms wiederhergestellt. Sie ist auch über das Tastaturkürzel Strg + T zu erreichen.

Verbesserungen

  • Das Tastaturkürzel Strg + S und die Funktion Kartei > Speichern rufen den Speicherbefehl nun auch für das Notizen-Fenster und die Karteikarte auf. Verläuft das Speichern der Notizen bzw. des Belegs fehlerfrei, wird direkt im Anschluss die Kartei gespeichert.
  • Meldungen, dass keine Änderungen vorgenommen wurden und ein Speichern unnötig sei, wurden entfernt.
  • Belegliste: Die Maximalbreite der Absätze wurde leicht erhöht.
  • DTA-Import: Beim Import mehrerer Seiten wird der Seiten- oder Spaltenumbruch im Texte nun markiert. Angegeben wird immer die Nummer der folgenden Seite/Spalte, und zwar nach diesem Schema: "[:SEITE:]". Diese Markierung der Seitenangabe wird mit den Schaltern, die bisher nur zum Ein- bzw. Ausblenden der Silbentrennung dienten, ein- bzw. ausgeblendet.
  • Karteikarte: Die Maximalbreite der Karteikarte wurde stark erhöht.
  • Dropdown-Menü: Einträge, die mehr als 65 Zeichen lang sind, haben jetzt immer einen Tooltip.
  • Dropdown-Menü: In mehrzeiligen Feldern werden auch dann sinnvolle Vorschläge geliefert, wenn bereits ein Wert im Textfeld steht.
  • Dropdown-Menü: Erscheint das Dropdown-Menü während der Texteingabe, werden bereits im Feld vorhandene Werte nicht mehr zum Hinzufügen angeboten.
  • Dropdown-Menü: Wird infolge einer Texteingabe ein Wert aus einer gefilterten Vorschlagsliste ausgewählt, wird automatisch die Zeile überschrieben, in der begonnen wurde zu tippen.
  • Dropdown-Menü: Versehntliches Einfügen von Leerzeilen zwischen den Werten wird in mehrzeiligen Feldern von nun an vermieden.

Behobene Fehler

  • DTA-Import: Der Import der letzten Seite eines Werks war mitunter nur unvollständig.
  • Karteikarte: Wenn die Felder mit den Feldüberschriften sehr schmal wurden, konnten sich Text und Kopier-Icon unter Umständen überlappen.
  • Notizen-Fenster: Unter Umständen wurde man zweimal gefragt, ob die Notizen gelöscht werden sollen.
  • Dropdown-Menü: Unter Windows konnten Menü-Einträge nicht mit der Maus ausgewählt werden.
  • Dropdown-Menü: Die Dialoge waren dysfunktional, falls in einem der Datenfelder des Belegs bereits Text stand.
  • Dropdown-Menü: Wurden in der Karteikarte die Felder Bedeutung oder Textsorte über das Dropdown-Menü gefüllt, wurden die Werte beim Speichern nicht übernommen.
Version 0.2.09. April 2019

Verbesserungen

  • Suchfunktion: Das Finden von Wörtern mit Sonderzeichen (ẽ und ñ) wurde verbessert.
  • Belegtext: Beim Kopieren wird die Quellenangabe automatisch mitkopiert. Dies betrifft alle Kopierfunktionen: das Kopieren via Icon und das Kopieren einer Textauswahl via Kontextmenü; beides sowohl in der Belegliste als auch in der Karteikarte.
  • Karteikarte: Die Texttools erkennen illegales Nesting jetzt in mehr Fällen als zuvor.

Behobene Fehler

  • Karteikarte: Das Einfügen von Text im Belegfeld durch Klick auf das Einfüge-Icon funktionierte nicht.
  • Karteikarte: Beim Einfügen von Text im Belegfeld durch Klick auf das Einfüge-Icon wurden einfache Zeilenumbrüche im HTML-Quelltext ignoriert.
  • Karteikarte: Ein Hinweis auf illegales Nesting erschien nicht in jedem Fall.
